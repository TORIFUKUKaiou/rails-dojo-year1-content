## 1. Rubyの基本パーツを振り返る

はじめに、これまでに学んだ基本パーツを、短いコードで順番に振り返ります。

### 1-1. 変数と代入

変数は、値に名前をつけて保存しておくためのものです。

Rubyの `=` は、数学で使う「等しい」という意味ではありません。右側の値を左側の変数に入れる**代入**を表します。

```ruby
total = 100
total = total + 50

puts total
```

> [!IMPORTANT]
> `total = total + 50` では、現在の `total` の値である `100` に `50` を足しています。その計算結果である `150` を、もう一度 `total` に代入しています。

#### 考えてみよう

このコードを実行すると、画面には何が表示されるでしょうか。

<details>
<summary>解答例を見る</summary>

```text
150
```

最初に `total` へ `100` を代入します。次に、`100 + 50` の計算結果である `150` を、もう一度 `total` へ代入しています。

</details>

---

### 1-2. 条件分岐

条件分岐は、条件によって実行する処理を切り替える仕組みです。

```ruby
score = 75

if score >= 80
  puts "よくできました！"
elsif score >= 60
  puts "合格です"
else
  puts "もう一度挑戦しましょう"
end
```

上から順番に条件を調べ、最初に条件が成り立った部分の処理を実行します。

#### 考えてみよう

- `score` が `75` のとき、何が表示されるでしょうか。
- `score` を `50` に変更すると、表示はどう変わるでしょうか。

<details>
<summary>解答例を見る</summary>

`score` が `75` のときは、次のように表示されます。

```text
合格です
```

`75 >= 80` は成り立ちません。しかし、`75 >= 60` は成り立つため、`elsif` の中の処理が実行されます。

`score` を `50` に変更すると、次のように表示されます。

```text
もう一度挑戦しましょう
```

どちらの条件も成り立たないため、`else` の中の処理が実行されます。

</details>

---

### 1-3. 繰り返しと配列

配列は、複数の値を順番にまとめて管理するためのものです。

`each` を使うと、配列の要素を先頭から1つずつ取り出して、同じ処理を繰り返せます。

```ruby
numbers = [3, 5, 7]

numbers.each do |number|
  puts number * 2
end
```

このコードでは、配列から取り出した値が、繰り返すたびに `number` に代入されます。

#### 考えてみよう

このコードを実行すると、何が何行に分かれて表示されるでしょうか。

<details>
<summary>解答例を見る</summary>

次のように、3行に分かれて表示されます。

```text
6
10
14
```

配列から `3`、`5`、`7` を順番に取り出し、それぞれを2倍しています。

</details>

---

### 1-4. 範囲オブジェクトと繰り返し

`1..5` のように、始まりの値と終わりの値を `..` でつなぐと、連続した値の範囲を表せます。

このような値を**範囲オブジェクト**と呼びます。

```ruby
range = 1..5

range.each do |number|
  puts number
end
```

ここでは、範囲オブジェクトを変数 `range` に代入してから、`each` を使っています。

#### 考えてみよう

- このコードを実行すると、何が表示されるでしょうか。
- `range = 3..7` に変更すると、表示はどう変わるでしょうか。

<details>
<summary>解答例を見る</summary>

最初のコードでは、次のように表示されます。

```text
1
2
3
4
5
```

`range = 3..7` に変更すると、次のように表示されます。

```text
3
4
5
6
7
```

`..` を使った範囲には、始まりの値と終わりの値の両方が含まれます。

</details>

---

### 1-5. ハッシュ

ハッシュは、値を「キー」と組み合わせて管理するためのものです。

```ruby
user = { "name" => "佐藤", "age" => 19 }

puts user["name"]
```

この例では、`"name"` と `"age"` がキーです。それぞれのキーを使って、対応する値を取り出せます。

> [!NOTE]
> `=>` は、ハッシュのキーと値を組み合わせる記号です。**ハッシュロケット**とも呼ばれます。

#### 考えてみよう

- このコードを実行すると、何が表示されるでしょうか。
- 年齢の `19` を表示するには、`puts` の行をどのように書けばよいでしょうか。

<details>
<summary>解答例を見る</summary>

最初のコードでは、次のように表示されます。

```text
佐藤
```

年齢を表示するには、キー `"age"` を使います。

```ruby
puts user["age"]
```

実行すると、次のように表示されます。

```text
19
```

</details>

---

### 1-6. メソッド

メソッドは、ひとまとまりの処理に名前をつけたものです。

```ruby
def double(number)
  number * 2
end

result = double(5)

puts result
```

`double(5)` を実行すると、引数の `5` が `number` に代入されます。

メソッドの最後に評価された `number * 2` の結果が、メソッドの呼び出し元へ返されます。この返された値を**戻り値**と呼びます。

#### 考えてみよう

- このコードを実行すると、何が表示されるでしょうか。
- `double(10)` を実行した場合、戻り値はいくつになるでしょうか。

<details>
<summary>解答例を見る</summary>

最初のコードでは、次のように表示されます。

```text
10
```

`double(5)` の中では、`5 * 2` が計算されます。その結果である `10` が戻り値です。

`double(10)` の戻り値は `20` です。

</details>

---

### 1-7. クラスとインスタンス

クラスは、データと処理をまとめるための設計図です。

クラスをもとに作られた1つ1つの実体を、インスタンスと呼びます。

```ruby
class Dog
  def initialize(name)
    @name = name
  end

  def bark
    "#{@name}はワンと鳴いた"
  end
end

pochi = Dog.new("ポチ")

puts pochi.bark
```

`Dog.new("ポチ")` を実行すると、`Dog` クラスから新しいインスタンスが作られます。そのとき、`initialize` メソッドが呼び出され、受け取った `"ポチ"` が `@name` に代入されます。

`@name` のように `@` から始まる変数を、**インスタンス変数**と呼びます。インスタンス変数に保存した値は、同じインスタンスの別のメソッドからも使えます。

#### 考えてみよう

- このコードを実行すると、何が表示されるでしょうか。
- `"ポチ"` ではなく `"ハチ"` と表示するには、どこを変更すればよいでしょうか。

<details>
<summary>解答例を見る</summary>

最初のコードでは、次のように表示されます。

```text
ポチはワンと鳴いた
```

`"ハチ"` と表示するには、`Dog.new` に渡す文字列を変更します。

```ruby
pochi = Dog.new("ハチ")
```

変数名も犬の名前に合わせるなら、次のように書けます。

```ruby
hachi = Dog.new("ハチ")

puts hachi.bark
```

</details>

## 2. パーツを組み立てる ── お店のレジプログラム

ここからは、これまでに学んだパーツを組み合わせて、少しずつプログラムを発展させます。

題材は、お店の注文金額を計算するプログラムです。

### ステップ1：1つの商品の金額を計算する

最初は、カレーを1つ注文するプログラムから始めます。

商品名と価格はハッシュで管理します。大盛りを選んだ場合は、通常価格に100円を追加します。

```ruby
item = { "name" => "カレー", "price" => 700 }
is_large = true

total = item["price"]

if is_large
  total = total + 100
end

puts "#{item["name"]}の合計金額は#{total}円です"
```

このプログラムでは、次のパーツを組み合わせています。

- ハッシュで、商品名と価格を管理する
- 変数に、現在の合計金額を保存する
- `if` で、大盛り料金を追加するか判断する
- `#{}` で、文字列の中に値を埋め込む

実行結果は次のとおりです。

```text
カレーの合計金額は800円です
```

#### 考えてみよう

`is_large` を `false` に変更すると、合計金額はいくらになるでしょうか。

<details>
<summary>解答例を見る</summary>

合計金額は `700` 円です。

`is_large` が `false` の場合、`if` の中の処理は実行されません。そのため、大盛り料金の100円は追加されません。

```text
カレーの合計金額は700円です
```

</details>

---

### ステップ2：複数の商品の合計を計算する

次は、複数の商品を注文できるようにします。

商品を表すハッシュを配列に入れ、`each` で1つずつ取り出します。

```ruby
items = [
  { "name" => "カレー", "price" => 700 },
  { "name" => "サラダ", "price" => 300 },
  { "name" => "ジュース", "price" => 200 }
]

total = 0

items.each do |item|
  puts "#{item["name"]}：#{item["price"]}円"
  total = total + item["price"]
end

puts "合計金額は#{total}円です"
```

`items` は配列です。その中に、商品を表すハッシュが3つ入っています。

`each` で商品を1つずつ `item` に取り出し、価格を `total` に足しています。

実行結果は次のとおりです。

```text
カレー：700円
サラダ：300円
ジュース：200円
合計金額は1200円です
```

#### 考えてみよう

`items` に次の商品を追加すると、合計金額はいくらになるでしょうか。

```ruby
{ "name" => "ケーキ", "price" => 400 }
```

<details>
<summary>解答例を見る</summary>

合計金額は `1600` 円です。

```text
カレー：700円
サラダ：300円
ジュース：200円
ケーキ：400円
合計金額は1600円です
```

配列には、次のように追加できます。

```ruby
items = [
  { "name" => "カレー", "price" => 700 },
  { "name" => "サラダ", "price" => 300 },
  { "name" => "ジュース", "price" => 200 },
  { "name" => "ケーキ", "price" => 400 }
]
```

</details>

---

### ステップ3：クラスとメソッドにまとめる

最後に、注文のデータと合計金額を計算する処理を、`Order` クラスにまとめます。

```ruby
class Order
  def initialize(items)
    @items = items
  end

  def total_price
    total = 0

    @items.each do |item|
      total = total + item["price"]
    end

    total
  end
end

items = [
  { "name" => "カレー", "price" => 700 },
  { "name" => "サラダ", "price" => 300 },
  { "name" => "ジュース", "price" => 200 }
]

order = Order.new(items)

puts "合計金額は#{order.total_price}円です"
```

このプログラムでは、次のパーツがつながっています。

- **ハッシュ**で、1つの商品を表す
- **配列**で、複数の商品をまとめる
- **インスタンス変数**で、注文の情報を保存する
- **`each`**で、商品を1つずつ取り出す
- **変数**で、計算の途中の値を保存する
- **メソッド**で、合計金額を計算する処理をまとめる
- **クラス**で、注文のデータと処理を1つにまとめる

最初は別々に学んだパーツも、組み合わせることで、1つの目的を持ったプログラムになります。

> [!IMPORTANT]
> プログラムが大きくなっても、使っている基本パーツが急に別のものへ変わるわけではありません。変数、条件分岐、繰り返し、配列、ハッシュ、メソッド、クラスを、目的に合わせて組み合わせていきます。

## 3. Ruby on Railsへの架け橋

第10回以降は、<ruby>Ruby on Rails<rt>ルビーオンレイルズ</rt></ruby>を使ったWebアプリケーションの開発へ進みます。

Railsのコードには、まだ学んでいない書き方も登場します。しかし、その土台には、これまでに学んだRubyが使われています。

ここではコードをすべて理解しようとせず、見覚えのあるパーツを探してみましょう。

### コントローラの例

コントローラは、Webアプリケーションの中で処理を担当する場所です。

```ruby
class ArticlesController < ApplicationController
  def index
    @articles = Article.all
  end
end
```

このコードには、これまでに学んだ次のパーツが登場しています。

- `class` でクラスを定義している
- `def` で `index` メソッドを定義している
- `@articles` というインスタンス変数に値を代入している

`< ApplicationController` や `Article.all` の意味は、これから学びます。

今は、RailsのコードにもRubyのクラス、メソッド、代入、インスタンス変数が使われていることを確認できれば十分です。

---

### ビューの例

ビューは、ブラウザに表示する画面を作る場所です。

```erb
<h1>記事一覧</h1>

<% @articles.each do |article| %>
  <p><%= article.title %></p>
<% end %>
```

このコードは、HTMLの中にRubyを組み合わせて書いています。

ここにも、これまでに学んだパーツが登場しています。

- `@articles` というインスタンス変数を使っている
- `each` で記事を1つずつ取り出している
- `do` と `end` で、繰り返す処理の範囲を表している

`<% %>` や `<%= %>` は、ビューの中でRubyを使うための書き方です。詳しい使い方は、これからRailsの授業で学びます。

> [!NOTE]
> 今はRailsのコードを自分で書けなくても問題ありません。
>
> 「Rubyで学んだ `class`、`def`、`@`、`each` が、Webアプリケーションの中にも登場する」と分かれば、この予告編のゴールは達成です。
