## 1. Ruby基本パーツのクイック復習と短い確認

これまでに学んだ6つの基本パーツを、短いコードで順番におさらいします。

### ① 変数と代入
変数は、データに名前をつけて保存する「箱」です。`=` は等しいという意味ではなく、**「右の値を左の箱に入れる（代入する）」**という意味でした。

```ruby
total = 100
total = total + 50
puts total
```

> [!IMPORTANT]
> `total = total + 50` は、「現在の `total`（100）に 50 を足した結果（150）を、再び `total` という箱に入れ直す」という処理です。

#### 💻 短い確認
上記のコードを実行すると、画面には何が表示されるでしょうか？

---

### ② 条件分岐（if）
`if` は、「もし〇〇なら、この処理をする」と動きを切り替える仕組みです。

```ruby
score = 75

if score >= 80
  puts "よくできました！"
elsif score >= 60
  puts "合格です"
else
  puts "がんばりましょう"
end
```

#### 💻 短い確認
`score` が `75` のとき、画面には何が表示されるでしょうか？また、`score` の値を `50` に書き換えて実行すると、表示はどう変わるでしょうか？

---

### ③ 繰り返しと配列
配列は、「順番のあるデータの集まり」です。`each` を使うと、配列の中身を先頭から順番に1つずつ取り出して繰り返し処理を行うことができます。

```ruby
numbers = [3, 5, 7]

numbers.each do |num|
  puts num * 2
end
```

> [!TIP]
> 初学者には、`(1..10).each` のような範囲指定よりも、まず配列や範囲を変数に入れてから `each` を使う書き方が見やすくておすすめです。
> ```ruby
> range = 1..5
> range.each do |i|
>   puts i
> end
> ```

#### 💻 短い確認
上記の `numbers.each` のコードを実行すると、画面には何が表示されるでしょうか？（何行に分かれて表示されるかも考えてみましょう）

---

### ④ ハッシュ
ハッシュは、データを「番号」ではなく「名前（キー）」で管理する仕組みです。

```ruby
user = { "name" => "佐藤", "age" => 19 }
puts user["name"]
```

> [!NOTE]
> `=>` は**ハッシュロケット**と呼び、左側のキーと右側の値を結びつけます。

#### 💻 短い確認
上記のコードを実行すると、画面には何が表示されるでしょうか？また、佐藤さんの年齢（`19`）を画面に表示させたい場合、2行目の `puts` のあとをどのように書き換えればよいでしょうか？

---

### ⑤ メソッド
メソッドは、「処理（やること）」に名前をつけて、いつでも呼び出せるようにまとめたものです。

```ruby
def double(number)
  number * 2
end

result = double(5)
puts result
```

> [!IMPORTANT]
> メソッドの中で一番最後に計算された結果が、メソッドの「戻り値（結果）」として外側に返されます。

#### 💻 短い確認
上記のコードを実行すると、画面には何が表示されるでしょうか？また、`double(10)` を呼び出した場合、結果はどうなるでしょうか？

---

### ⑥ クラスとインスタンス
クラスはデータと処理をまとめた**「設計図」**であり、そこから作られた実物を**「インスタンス」**と呼びます。

```ruby
class Dog
  def initialize(name)
    @name = name
  end

  def bark
    "#{@name}はワンと鳴いた"
  end
end

pochi = Dog.new("ポチ")
puts pochi.bark
```

> [!IMPORTANT]
> - `Dog.new` をすると、クラスの `initialize` メソッドが自動的に呼び出されます。
> - `@name` のように `@` で始まる変数は**インスタンス変数**と呼ばれ、そのインスタンスの中だけでずっとデータを覚えておくことができます。

#### 💻 短い確認
上記のコードを実行すると、画面には何が表示されるでしょうか？また、名前を `"ハチ"` に変えて鳴かせるには、コードのどこを書き換えればよいでしょうか？

## 2. パーツを組み立てる ── お店のレジプログラム

それでは、これまで学んだパーツを少しずつ組み合わせて、段階的により大きなプログラムを作ってみましょう。今回は「お店のレジプログラム」を題材にします。

### ステップ1：変数と条件分岐だけ（まずは大盛り判定から）
まずは、カレー（700円）を注文し、「大盛りにするかどうか」で合計金額を計算するシンプルなプログラムです。

```ruby
total = 700
is_large = true # 大盛りにするかどうか

if is_large
  total = total + 100 # 大盛りはプラス100円
end

puts "合計金額：#{total}円"
```

ここでは、変数（`total`, `is_large`）と条件分岐（`if`）を組み合わせて、大盛りの場合だけ合計金額を上書き更新しています。

---

### ステップ2：配列と `each` を加える（複数商品の合計計算）
次に、複数の商品を同時に買ったときの合計金額を、配列と `each` を使って計算できるように進化させます。

```ruby
# 注文した商品の価格一覧
prices = [700, 500, 300]
total = 0

prices.each do |price|
  total = total + price
end

puts "合計金額：#{total}円"
```

配列 `prices` の中身を `each` で1つずつ取り出し、`total` に足し合わせていくことで、何点商品があっても自動で合計を計算できるようになりました。

---

### ステップ3：クラスとメソッドにまとめる（レジ機能の整理）
最後に、この「注文の合計を計算する」というデータと処理を、クラスとメソッドを使ってきれいに整理します。

```ruby
class Order
  # 注文を作るときに、価格一覧を受け取って保存する
  def initialize(prices)
    @prices = prices
  end

  # 合計金額を計算して返すメソッド
  def total_price
    total = 0
    @prices.each do |price|
      total = total + price
    end
    total # 最後に合計金額を戻り値として返す
  end
end

# 1. 注文インスタンスを作る
my_order = Order.new([700, 500, 300])

# 2. 合計金額を計算させて表示する
puts "合計金額：#{my_order.total_price}円"
```

このように、クラス（`Order`）の中にデータ（`@prices`）と処理（`total_price` メソッド）をまとめることで、すっきりとしたプログラムになりました。

プログラムが大きくなっても、この「クラス」という設計図を準備しておくことで、簡単に新しい注文を作って合計を計算できるようになります。

## 3. Railsへの架け橋 ── 学んだことがどう繋がるか？

来週以降、いよいよ <ruby>Ruby on Rails<rt>ルビーオンレイルズ</rt></ruby> を使った Webアプリケーションの開発が始まります。

「これまで学んだ黒い画面（ターミナル）のプログラムは、Webアプリ開発にどう繋がるんだろう？」と疑問に思うかもしれません。実は、Railsのプログラムの中身は、今みんなが復習したRubyの書き方そのものです。

ちょっとだけ、Railsで実際に書くコードの「予告編」を見てみましょう。

### コントローラ（処理を交通整理するファイル）の例
```ruby
# Railsのコントローラの例（形を眺めるだけでOKです！）
class ArticlesController
  def index
    # 記事の一覧を表示する処理をここに書く
  end
end
```
ほら、`class` と `def`（メソッド）が使われていますね！これはまさに、これまでの授業で作ってきたクラスやメソッドと同じ構造です。

### ビュー（画面に表示するファイル）の例
```html
<!-- Railsのビューの例（形を眺めるだけでOKです！） -->
<h1>記事一覧</h1>

<% @articles.each do |article| %>
  <p><%= article.title %></p>
<% end %>
```
ここでは、`@articles` という `@`（インスタンス変数）からデータを取り出し、`each` を使って繰り返し画面に表示しています。これも、Rubyで学んだ「インスタンス変数」と「配列の `each` 繰り返し」の知識がそのまま使われています。

> [!NOTE]
> データベースとの連携や、`class ArticlesController < ApplicationController` の `<`（継承）など、詳しい仕組みは第10週以降や後期にゆっくり学んでいきます。
> 今は、**「詳しい仕組みは第10週以降に学ぶので、今はRubyで見た記述がRailsにも登場することだけ確認しましょう」**。
> **「これまでRubyで書いてきた `@` や `def`、`each` などのパーツが、そのままRailsのWebアプリでも大活躍するんだ！」**という雰囲気を感じられればバッチリです。
