# 第6回：ハッシュ ── 名前でデータを管理する

## 今日のゴール

ハッシュを使って、データを「番号」ではなく「名前」で取り出せるようになる。

配列とハッシュを組み合わせた形にも触れます。Webサービスのデータや<ruby>JSON<rt>ジェイソン</rt></ruby>では、ハッシュの中に配列が入ったり、ハッシュの中にハッシュが入ったりする形がよく出てきます。

---

## 前回のおさらい

前回は、配列を使ってたくさんのデータをまとめて持ちました。

```ruby
foods = ["カレー", "ラーメン", "寿司"]

puts foods[0]
puts foods[1]
```

配列は、「順番」でデータを管理します。

- `foods[0]` → 1番目
- `foods[1]` → 2番目

たくさんのデータを順番に並べたいときは、配列が便利です。

```ruby
scores = [80, 55, 100]

puts scores.sum
puts scores.length
```

ただし、1人分のプロフィールのように、「名前」「年齢」「出身地」のような意味の違うデータをまとめたいときは、番号だけでは分かりにくくなります。

今日は、番号ではなく「名前」でデータを取り出すハッシュを学びます。

---

## なぜハッシュが必要なのか

たとえば、1人分のプロフィールを持ちたいとします。

```ruby
name = "田中"
age = 20
city = "福岡"
```

これでも動きます。

ただし、これらは「1人分の情報」としてまとまっているわけではありません。

こういうときに使うのがハッシュです。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }
```

これで、`name`、`age`、`city` を1つにまとめて持てます。

![](../images/array-vs-hash-access.svg)

---

## ハッシュとは

ハッシュは、「キー」と「値」をセットで持つデータです。

```ruby
person = { "name" => "田中", "age" => 20 }
```

この場合：

- `"name"` がキー
- `"田中"` が値
- `"age"` がキー
- `20` が値

キーは、値を取り出すための名前です。

配列は番号で取り出します。

```ruby
foods = ["カレー", "ラーメン", "寿司"]

puts foods[0]
```

ハッシュはキーで取り出します。

```ruby
person = { "name" => "田中", "age" => 20 }

puts person["name"]
```

---

## ハッシュの書き方

書き方のポイントは次のとおりです。

- `{}` を使う
- `=>` でキーと値をつなぐ
- `,` で区切る

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }
```

長くなる場合は、複数行に分けて書いても構いません。

```ruby
person = {
  "name" => "田中",
  "age" => 20,
  "city" => "福岡"
}
```

どちらも同じハッシュです。

---

## キーを使って値を取り出す

ハッシュの中身は、キーを使って取り出します。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }

puts person["name"]
puts person["age"]
puts person["city"]
```

実行すると：

```text
田中
20
福岡
```

`person["name"]` は、「`person` の中から `name` というキーの値を取り出す」という意味です。

---

## 文章の中で使う

ハッシュから取り出した値は、文字列の中でも使えます。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }

puts "#{person["name"]}さんは#{person["city"]}出身です"
```

実行すると：

```text
田中さんは福岡出身です
```

これまで学んだ `#{}` と同じです。

---

## 値を書き換える・追加する

ハッシュは、あとから値を書き換えたり、新しい情報を追加したりできます。

```ruby
person = { "name" => "田中", "age" => 20 }

person["age"] = 21
person["city"] = "福岡"

puts person["age"]
puts person["city"]
```

この場合：

- `age` は `20` から `21` に書き換わる
- `city` は新しく追加される

配列で `foods[1] = "うどん"` と書いたときと考え方は似ています。

配列は番号を指定して書き換えます。

```ruby
foods[1] = "うどん"
```

ハッシュはキーを指定して書き換えます。

```ruby
person["age"] = 21
```

---

## `each` で全部見る

ハッシュも `each` で1つずつ見られます。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }

person.each do |key, value|
  puts "#{key}: #{value}"
end
```

実行すると：

```text
name: 田中
age: 20
city: 福岡
```

配列の `each` では、1つの値を取り出しました。

```ruby
foods.each do |food|
  puts food
end
```

ハッシュの `each` では、キーと値をセットで取り出します。

```ruby
person.each do |key, value|
  puts "#{key}: #{value}"
end
```

---

## ハッシュの便利メソッド

ハッシュにも、便利なメソッドがあります。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }

p person.keys
p person.values
puts person.length
puts person.key?("name")
```

それぞれの意味は次のとおりです。

- `keys`：キーの一覧を取り出す
- `values`：値の一覧を取り出す
- `length`：キーと値のセットがいくつあるか調べる
- `key?("name")`：そのキーがあるか調べる

`person.keys` は、キーの一覧を配列として返します。

`person.values` も、値の一覧を配列として返します。

`puts person.keys` と書くと、キーが1行ずつ表示されます。

```text
name
age
city
```

配列の形で確認したいときは、`p` を使います。

```ruby
p person.keys
p person.values
```

実行すると、配列の形のまま表示できます。

```text
["name", "age", "city"]
["田中", 20, "福岡"]
```

`p` は、値の中身を確認したいときに使える表示用の命令です。

`key?` は、条件分岐と組み合わせると便利です。

```ruby
person = { "name" => "田中", "age" => 20 }

if person.key?("city")
  puts person["city"]
else
  puts "出身地はまだ登録されていません"
end
```

---

## 配列とハッシュの違い

配列とハッシュは、どちらもデータをまとめる道具です。

違いは、何で取り出すかです。

| 種類 | 向いていること | 取り出し方 |
|---|---|---|
| 配列 | 順番に並んだデータ | `foods[0]` |
| ハッシュ | 名前のついたデータ | `person["name"]` |

たとえば、好きな食べ物を3つ持つなら配列が向いています。

```ruby
foods = ["カレー", "ラーメン", "寿司"]
```

1人分のプロフィールを持つならハッシュが向いています。

```ruby
person = { "name" => "田中", "age" => 20, "city" => "福岡" }
```

---

## 配列の中にハッシュを入れる

1人分の情報は、ハッシュで表せます。

```ruby
student = { "name" => "田中", "score" => 80 }
```

複数人分の情報は、配列の中にハッシュを入れると表せます。

```ruby
students = [
  { "name" => "田中", "score" => 80 },
  { "name" => "鈴木", "score" => 65 },
  { "name" => "佐藤", "score" => 92 }
]
```

`each` で1人ずつ取り出せます。

```ruby
students.each do |student|
  puts "#{student["name"]}: #{student["score"]}点"
end
```

この形は、実際のアプリでもよく出てきます。

---

## ハッシュの値に配列を入れる

ハッシュの値には、文字列や数値だけでなく、配列を入れることもできます。

```ruby
person = {
  "name" => "田中",
  "favorites" => ["カレー", "ラーメン", "寿司"]
}
```

好きな食べ物の一覧を取り出すには、次のように書きます。

```ruby
puts person["favorites"][0]
puts person["favorites"][1]
```

実行すると：

```text
カレー
ラーメン
```

`person["favorites"]` で配列を取り出し、その配列の `0` 番目、`1` 番目を見ています。

---

## ハッシュの値にハッシュを入れる

ハッシュの値に、さらにハッシュを入れることもできます。

```ruby
student = {
  "name" => "田中",
  "scores" => {
    "japanese" => 80,
    "math" => 90,
    "english" => 75
  }
}
```

数学の点数を取り出すには、次のように書きます。

```ruby
puts student["scores"]["math"]
```

実行すると：

```text
90
```

`student["scores"]` で点数のハッシュを取り出し、その中から `"math"` を取り出しています。

---

## <ruby>JSON<rt>ジェイソン</rt></ruby>でよく出る形

Webサービスでは、<ruby>JSON<rt>ジェイソン</rt></ruby>というデータ形式がよく使われます。

<ruby>JSON<rt>ジェイソン</rt></ruby>とRubyのハッシュはまったく同じ書き方ではありませんが、「名前をつけて値を持つ」「配列やハッシュを入れ子にする」という考え方は似ています。

たとえば、アプリで使うデータは次のような形になることがあります。

```ruby
article = {
  "title" => "はじめての記事",
  "tags" => ["Ruby", "Rails"],
  "author" => {
    "name" => "田中",
    "city" => "福岡"
  }
}
```

タイトルを取り出す：

```ruby
puts article["title"]
```

1つ目のタグを取り出す：

```ruby
puts article["tags"][0]
```

作者の名前を取り出す：

```ruby
puts article["author"]["name"]
```

入れ子の形は最初は難しく見えます。
ただし、1つずつ見ると、配列とハッシュを組み合わせているだけです。

## シンボルのキー

Rubyでは、キーに文字列ではなくシンボルを使う書き方もあります。

```ruby
person = { name: "田中", age: 20, city: "福岡" }

puts person[:name]
```

この授業の練習では、まず文字列のキーを使います。

```ruby
person = { "name" => "田中", "age" => 20 }

puts person["name"]
```

ただし、RubyやRailsではシンボルのキーもよく出てきます。見かけたときに、これもハッシュのキーだと分かればOKです。
