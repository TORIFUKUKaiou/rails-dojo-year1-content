# Stretch：Webの仕組みを固めて、Rubyの総復習へ進もう

この Stretch では、第10回の Practice で確認したWebの基本をもう一度固めます。

そのあと、第9回の発展練習を使って、Rubyの文法を総復習します。

---

## 取り組み方

次の順番で進めてください。

1. 第10回 Practice のWeb教材を開く
2. 満点になるまで繰り返す
3. 間違えた問題の解説を読み直す
4. 満点になったら、第9回 Stretch に進む
5. Rubyの総復習として、できるところまで問題を解く

---

## 1. 第10回 Practice を満点になるまで繰り返す

第10回 Practice のWeb教材を開いてください。

公開URL：

```text
https://TORIFUKUKaiou.github.io/rails-dojo-web-basics-quiz/
```

選択肢を選び、`submit` を押して、解答と解説を確認してください。

最後の結果画面で、全問正解になるまで繰り返します。

> [!IMPORTANT]
> 間違えた問題は、すぐに次へ進まず、解説を読んでください。
> とくに、リクエストとレスポンス、GETとPOST、フォームの `action` と `method`、ルーティングの読み方を確認してください。

### 確認すること

満点になったら、次の言葉を自分の言葉で説明できるか確認してください。

- ブラウザ
- サーバ
- リクエスト
- レスポンス
- URL
- GET
- POST
- フォーム
- ルーティング

説明に迷う言葉があれば、もう一度Web教材の該当問題を確認してください。

---

## 2. 第9回 Stretch でRuby文法を総復習する

第10回は、Railsに入る前のWebの仕組みを確認する回です。

次回からRailsアプリケーションの作成に入るため、Rubyの基本文法も復習しておきます。

次の教材へ進んでください。

- [第9回 Stretch：Rubyのパーツを組み合わせて、さらに先へ進もう](../09/stretch.md)

第9回 Stretch では、次の内容をまとめて復習できます。

- 計算と条件分岐
- 配列とハッシュのデータ処理
- メソッドで処理を整理する
- クラスで状態を管理する
- 検索や生成AIも使うチャレンジ課題

問題ごとに `stretch01.rb`、`stretch02.rb` のようなファイルを作り、実行結果を確認しながら進めてください。

```bash
ruby stretch01.rb
```

表示例がある問題では、実行結果と表示例を比べてください。

解答例を見る前に、まず自分でコードを書いて実行します。
動いたあとで解答例を開き、自分のコードとの違いを確認してください。

---

## 最後に確認すること

第10回のWeb教材と、第9回StretchのRuby復習は、次回以降のRails学習につながります。

Railsでは、ブラウザから届いたリクエストをルーティングで受け取り、Rubyのコードで処理を書きます。

次の2つを意識して復習してください。

- Webの流れを読めること
- Rubyのコードを自分で書いて実行できること
